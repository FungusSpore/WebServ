📄 About Prophet Web Server

🎯 Project Overview

Prophet Web Server is a custom HTTP/1.1 implementation built in C++ as part of 42KL Core Module 5. This project demonstrates advanced systems programming concepts including network programming, HTTP protocol implementation, and web server architecture.

🔧 Technical Features

  • HTTP/1.1 Protocol Support
  • Epoll-based Event Loop
  • CGI Script Execution
  • Cookie Session Management
  • File Upload Handling
  • Directory Auto-indexing
  • Custom Error Pages
  • URL Redirection

🏗️ Architecture

  • Non-blocking I/O Operations
  • Multi-server Configuration
  • Location-based Routing
  • Request/Response Pipeline
  • Memory-efficient Design
  • Configurable Limits
  • Error Recovery
  • Clean Resource Management

🛠️ Implementation Details

Core Components

WebServer: Main server orchestrator handling multiple server blocks and global configuration.

Socket & Epoll: Low-level networking with efficient event-driven I/O using Linux epoll.

MiniHttp: HTTP protocol parser and response builder with complete request/response handling.

CGI Handler: External script execution with environment variable passing and output processing.

Cookie System: Session management with secure cookie handling and user state tracking.

🔐 Security Features

  • Input Validation & Sanitization
  • Request Size Limits
  • HttpOnly Cookie Flags
  • Path Traversal Protection
  • Resource Access Controls
  • Error Information Limiting
